## Tech Debt Summary

### Warning Level (recommend addressing in v1.2)

1. **Edge fallback operator** — `drawCommunicationEdges.ts:150` has `?? 'support'` that never triggers for valid lineups but would mask bugs if instrument set expands
2. **No crash guard on malformed pair key** — `drawCommunicationEdges.ts:209-210` would throw TypeError on unrecognized instrument name from external input
3. **Single-read lineup pattern** — `VisualizerCanvas` captures lineup at mount only; correct given lock semantics but brittle if mid-session lineup changes are added

### Info Level (acceptable)

4. **App.tsx console.logs** — Present in one-shot calibration callback, not hot path; explicitly out of FIX-03 scope
5. **Visual confirmation needed** — Circular layout at 320px width (2 and 8 instruments) and dynamic edge hide transition need device testing on iOS

### Deferred to v1.2

6. **INST-08 / DISC-05** — Saxophone/keyboard disambiguation via chroma entropy; requires empirical calibration on real recordings

**Total:** 6 items across 4 phases (0 blockers, 3 warnings, 2 info, 1 deferred)
