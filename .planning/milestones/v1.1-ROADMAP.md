# Milestone v1.1: Flexible Lineup

**Status:** SHIPPED 2026-03-12
**Phases:** 9-11
**Total Plans:** 9

## Overview

Support any jazz combo instrument combination (2-8 instruments) with dynamic canvas layout, replacing the hardcoded 4-instrument limitation, plus v1.0 gap closures.

## Milestone Summary

**Key Decisions:**
- Circular layout over d3-force (deterministic, O(n), same visual result for uniform-weight graphs)
- InstrumentName kept as strict TypeScript union (not loosened to string) for compile-time safety
- Disambiguation logic (sax/keyboard, vibes/keyboard, multi-horn) deferred to v1.2
- Vibes + keyboard simultaneous selection prevented in UI (acoustically indistinguishable via FFT)
- Non-animated edges use rgba() instead of globalAlpha for single-pass batch rendering
- Pre-allocated edgeRenderBuf (28 slots) for zero per-frame heap allocations
- Bass-center convention: position[0] = center, ring starts at 12 o'clock
- iOS AudioContext must be created synchronously before any await in gesture handlers

**Issues Resolved:**
- Hardcoded 4-instrument limitation replaced with flexible 2-8 instrument support
- Diamond layout replaced with adaptive circular layout
- iOS loadExample AudioContext gesture bug fixed
- Dead code (InstrumentRoleOverlay) removed
- Hot-path console.logs removed
- Pocket line crashes on bass/drums-absent lineups fixed

**Issues Deferred:**
- INST-08 / DISC-05: Saxophone/keyboard disambiguation via chroma entropy (requires empirical calibration)
- Trombone vs bass disambiguation via onset timing
- Vibraphone vs keyboard disambiguation via tremolo modulation
- Horn section disambiguation (3+ horns)
- Instrument family color coding and spatial grouping

**Technical Debt Incurred:**
- drawCommunicationEdges.ts:150 retains `?? 'support'` fallback that never triggers for valid lineups
- drawCommunicationEdges.ts:209-210 lacks fallback for unknown pair keys (safe for 8 valid instruments)
- VisualizerCanvas captures lineup at mount time only (correct but structurally brittle)
- App.tsx retains console.logs in one-shot calibration callback (not hot path)
