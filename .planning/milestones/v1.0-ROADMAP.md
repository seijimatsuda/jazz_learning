# Milestone v1.0: Jazz Communication Visualizer MVP

**Status:** ✅ SHIPPED 2026-03-11
**Phases:** 1-8
**Total Plans:** 38

## Overview

The build proceeded as a strict dependency chain through the audio pipeline, then fanned out into parallel analysis modules, Canvas rendering, and React UI. The goal was a browser-native tool accurate enough that a jazz musician recognizes what is happening in the music by watching the visualization alone. Eight phases delivered that from scaffolding through advanced analysis features, with iOS Safari compatibility as a zero-tolerance constraint baked into Phase 1.

## Milestone Summary

**Key Decisions:**

See STATE.md for full decision log (183 decisions across 8 phases). Notable architectural decisions:
- Canvas API over D3.js — precise per-pixel control for glow/ripple animations
- Meyda.js for feature extraction — chroma, RMS, ZCR, spectral features out of the box
- Frequency band splitting (not stem separation) — browser-only, no backend
- iOS compatible from day one — AudioContext, Canvas, offscreen glow compositing
- HTMLCanvasElement (off-DOM) for glow layers — iOS 16 OffscreenCanvas support incomplete
- audioStateRef pattern — Web Audio objects non-serializable, rAF reads ref at 60fps
- All analysis at 10fps, visualization at 60fps rAF — clear separation of compute budgets

**Issues Resolved:**
- iOS AudioContext gesture requirement for file upload (fixed: pre-create in click handler)
- Chord name format (fixed: TYPE_DISPLAY map produces jazz notation like 'Cmaj7', 'G7')
- Tension staleness in React overlay (fixed: per-tick onTensionUpdate callback)
- InstrumentRoleOverlay visibility gap (fixed: component created, then superseded by Phase 5 canvas nodes)
- Meyda spectralFlux bug (workaround: hand-rolled computeSpectralFlux)
- iOS 16 OffscreenCanvas incompatibility (workaround: HTMLCanvasElement off-DOM)

**Issues Deferred:**
- iOS real-device 60fps verification (architecture correct, untested on hardware)
- loadExample iOS AudioContext gesture (broken: await before createAudioContext)
- INSTRUMENT_ORDER hardcoded to 4-instrument diamond (canvas ignores BandSetupPanel lineup)
- Example track audio file requires manual placement

**Technical Debt Incurred:**
- InstrumentRoleOverlay.tsx is dead code (159 lines, never rendered)
- console.log in AnalysisTick and CanvasRenderer hot paths
- getByteFrequencyData called 6x/frame in per-node loop (stale data for nodes 1-5)
- isCalibrated read from ref at render time (fragile, works via side effect)
