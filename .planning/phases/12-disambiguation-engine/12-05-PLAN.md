  <action>
Create `src/audio/DisambiguationEngine.ts` — the orchestrator that runs all applicable disambiguators.

**Function signature:**
```typescript
export function runDisambiguationEngine(
  instrs: InstrumentAnalysis[],
  freqData: Uint8Array,
  prevFreqData: Uint8Array,
  bands: FrequencyBand[],
  sampleRate: number,
  fftSize: number,
  state: DisambiguationState,
  chroma: number[] | null,
): void
```

**Algorithm:**

1. **Initialize displayActivityScore from rawActivityScore for all instruments:**
   ```typescript
   for (const instr of instrs) {
     instr.displayActivityScore = instr.rawActivityScore;
   }
   ```
   This ensures any instrument NOT touched by a disambiguator still gets a valid display score.

2. **Tutti guard (DISC-FND-04):**
   Check `isTuttiActive(instrs)` from instrumentFamilies.ts. If tutti:
   - Increment `state.tuttiFrameCount`
   - Set `state.isTutti = true`
   - Set all confidence values to 0 (uncertain during tutti)
   - Return early — no disambiguation during tutti (equal weights = displayActivityScore stays as rawActivityScore)

   If not tutti:
   - Reset `state.tuttiFrameCount = 0`
   - Set `state.isTutti = false`

3. **Trombone/bass pair (DISC-01, DISC-FND-05):**
   If `hasInstrumentPair(instrs, 'trombone', 'bass')`:
   - Call `disambiguateTromboneBass(freqData, bands, state)`
   - Apply weights: find trombone and bass in instrs, multiply `displayActivityScore *= weight`
   - Store confidence: `state.confidence['trombone_bass'] = confidence`

4. **Sax/keyboard pair (DISC-05, DISC-FND-05):**
   If `hasInstrumentPair(instrs, 'saxophone', 'keyboard')` and `chroma` is not null:
   - Call `disambiguateSaxKeyboard(chroma)`
   - Apply weights to sax and keyboard `displayActivityScore`
   - Store confidence: `state.confidence['sax_keyboard'] = confidence`

5. **Vibes/keyboard pair (DISC-02, DISC-FND-05):**
   If `hasInstrumentPair(instrs, 'vibes', 'keyboard')`:
   - Call `disambiguateVibesKeyboard(freqData, bands, state)`
   - Apply weights to vibes and keyboard `displayActivityScore`
   - Store confidence: `state.confidence['vibes_keyboard'] = confidence`
   - Note: if sax/keyboard already ran and modified keyboard's displayActivityScore, vibes/keyboard stacks on top. This is intentional — keyboard can be suppressed by multiple disambiguators, but only via displayActivityScore (rawActivityScore is untouched).

6. **Horn section (DISC-03, DISC-FND-05):**
   If `countHorns(instrs) >= 3`:
   - Collect horn instruments with their rawActivityScores
   - Call `disambiguateHornSection(freqData, horns, bands, sampleRate, fftSize)`
   - Apply weights to each horn's `displayActivityScore`
   - Store confidence: `state.confidence['horn_section'] = confidence`

7. **Existing kb/guitar disambiguation:** The existing KbGuitarDisambiguator already runs in AnalysisTick.ts before this engine and modifies `activityScore` directly. In this plan, we ensure that the kb/guitar weights are reflected in `displayActivityScore` by running this engine AFTER the existing kb/guitar block. The kb/guitar weights were already applied to `activityScore`, and since step 1 copies `rawActivityScore` (not `activityScore`) to `displayActivityScore`, we need to also apply the kb/guitar weights. Handle this by:
   - Checking if kb and guitar are both present
   - If so, re-apply the existing `activityScore` (which has kb/guitar weights) as the basis for their `displayActivityScore` instead of `rawActivityScore`
   - OR simpler: in AnalysisTick.ts, move the existing kb/guitar disambiguation to happen BEFORE `rawActivityScore` is set, so kb/guitar weights are baked into `rawActivityScore`. This is a design choice — but NO: the research says raw must be pre-disambiguation. So keep rawActivityScore clean and handle kb/guitar in this engine too.
   - **Best approach:** Leave the existing kb/guitar block in AnalysisTick.ts for now (it modifies `activityScore`), but in step 1 of this engine, set `displayActivityScore = activityScore` (not `rawActivityScore`). This captures kb/guitar weights that were already applied. The new disambiguators then stack on top.

   REVISED step 1:
   ```typescript
   for (const instr of instrs) {
     instr.displayActivityScore = instr.activityScore; // includes kb/guitar weights if applied
   }
   ```

Import all disambiguator functions, `hasInstrumentPair`, `countHorns`, `isTuttiActive` from instrumentFamilies.ts. Import types from types.ts.
