**Exported functions:**

1. `pushRmsSample(state: DisambiguationState, rms: number): void`
   Push current mid-band RMS into `state.tremoloRmsBuffer` ring buffer. Update `tremoloRmsHead` and `tremoloRmsSamples`. Exact implementation from research code examples.

2. `computeRmsVariance(state: DisambiguationState): number`
   Compute variance of the RMS ring buffer. Requires >= 5 samples. Return 0 if insufficient. Exact implementation from research code examples.

3. `disambiguateVibesKeyboard(freqData: Uint8Array, bands: FrequencyBand[], state: DisambiguationState): { vibesWeight: number; keyboardWeight: number; confidence: number }`

**Algorithm for disambiguateVibesKeyboard:**
1. Look up the 'mid' band from `bands` to get bin range. If not found, return equal weights.

2. Compute mid-band RMS from `freqData`:
   ```typescript
   let sumSq = 0;
   for (let i = midBand.lowBin; i <= midBand.highBin; i++) {
     sumSq += freqData[i] * freqData[i];
   }
   const rms = Math.sqrt(sumSq / (midBand.highBin - midBand.lowBin + 1));
   ```

3. Push RMS sample into state via `pushRmsSample(state, rms)`.

4. Compute RMS variance via `computeRmsVariance(state)`.

5. Map variance to vibes score:
   - High variance (> threshold) = probable tremolo = vibes-active
   - Threshold: `0.5` (CALIBRATION_NEEDED — this is a starting estimate)
   - `vibesScore = Math.min(1, variance / 1.0)` — variance of 1.0+ = fully vibes-like
   - Clamp to [0.15, 0.85]

6. **Confidence is capped at 0.5 maximum** — this is the hardest pair. At 10fps, tremolo detection is fundamentally probabilistic, not deterministic. Comment: `// MAX_CONFIDENCE = 0.5 — Nyquist limits tremolo detection at 10fps (see research)`

7. Confidence formula: `Math.min(0.5, (state.tremoloRmsSamples / 20) * Math.abs(vibesWeight - 0.5) * 2)`

8. Return `{ vibesWeight, keyboardWeight: complementary (clamped), confidence }`.

Import `DisambiguationState`, `FrequencyBand` from types.ts. Do NOT import from Meyda.

```typescript
export function disambiguateHornSection(
  freqData: Uint8Array,
  hornInstruments: Array<{ instrument: string; rawActivityScore: number }>,
  bands: FrequencyBand[],
  sampleRate: number,
  fftSize: number,
): { weights: Record<string, number>; confidence: number }
```

**Algorithm:**
1. Guard: if `hornInstruments.length < 3`, return empty weights with confidence 0.

2. Look up band range for horn centroid computation. Use combined range: 'mid' band lowBin to 'mid_high' band highBin (covering 250-3000 Hz where horn harmonics live). If bands not found, return equal weights.

3. Compute a single spectral centroid for the entire mix using `computeBandCentroid(freqData, lowBin, highBin, sampleRate, fftSize)`.

4. Assign weights based on expected centroid ordering: trombone < saxophone < trumpet.
   - Define expected centroid positions (Hz): `{ trombone: 900, saxophone: 1400, trumpet: 2250 }` — CALIBRATION_NEEDED
   - For each horn in the lineup, compute distance from actual centroid to expected position
   - Closer to expected position = higher weight for that instrument
   - Normalize weights so they sum to 1.0 across all horns

5. Apply minimum weight floor: each horn gets at least `0.15 / hornInstruments.length` so no horn is fully suppressed.

6. Confidence:
   - Start at 0.7
   - For each pair of horns: if their expected centroids are within 200 Hz, reduce confidence by 0.15
   - If any horn has rawActivityScore > 0.8, reduce confidence by 0.2 (dynamics shift centroid — see research pitfall #4)
   - Clamp confidence to [0, 1]

7. Return `{ weights, confidence }` where weights is a Record keyed by instrument name.
