Create `src/audio/TromboneBassDisambiguator.ts` implementing DISC-01.

**Function signature:**
```typescript
export function disambiguateTromboneBass(
  freqData: Uint8Array,
  bands: FrequencyBand[],
  state: DisambiguationState,
): { tromboneWeight: number; bassWeight: number; confidence: number }
```

**Algorithm:**
1. Look up the 'mid' band from `bands` to get bin range (250-2000 Hz). If not found, return equal weights (0.5/0.5) with confidence 0.

2. Compute spectral flatness of the mid band using `computeSpectralFlatness(freqData, midBand.lowBin, midBand.highBin)`.

3. Push flatness into `state.flatnessBuffer` ring buffer (update head, samples).

4. If fewer than 3 flatness samples accumulated, return equal weights with confidence 0 (insufficient data).

5. Compute average flatness from the ring buffer (last N samples where N = state.flatnessSamples, up to 10).

6. Map average flatness to disambiguation weight:
   - Higher flatness (> 0.3) = more trombone-like (trombone has more inharmonic mid content from slide positions)
   - Lower flatness (< 0.3) = more bass-like (bass is more tonal in the mid band)
   - Use linear interpolation: `tromboneScore = Math.min(1, avgFlatness / 0.6)` — flatness of 0.6+ = fully trombone-like
   - Clamp weights to [0.15, 0.85] matching existing convention

7. Compute confidence: `Math.min(1, state.flatnessSamples / 10) * Math.abs(tromboneWeight - 0.5) * 2`
   - Confidence rises as buffer fills and as weights diverge from equal

8. Return `{ tromboneWeight, bassWeight: 1 - tromboneScore (clamped), confidence }`.

**Mark thresholds (0.3, 0.6) with `// CALIBRATION_NEEDED` comments** per research open question #2.

Import `computeSpectralFlatness` from SpectralFeatures.ts. Import `FrequencyBand`, `DisambiguationState` from types.ts.

**Function signature:**
```typescript
export function disambiguateSaxKeyboard(
  chroma: number[],
): { saxWeight: number; keyboardWeight: number; confidence: number }
```

**Algorithm:**
1. Compute chroma entropy using `chromaEntropy(chroma)` from SpectralFeatures.ts.

2. Normalize entropy to [0, 1]: `normalizedEntropy = entropy / Math.log2(12)`.

3. Map normalized entropy to disambiguation weight:
   - Low entropy (< 0.3 normalized) = saxophone-like (monophonic, energy in few pitch classes)
   - High entropy (> 0.5 normalized) = keyboard-like (chords, energy spread across pitch classes)
   - Between 0.3 and 0.5: linear interpolation from sax-leaning to keyboard-leaning
   - `keyboardScore = (normalizedEntropy - 0.3) / 0.2` clamped to [0, 1], then mapped to weight range

4. Clamp weights to [0.15, 0.85]:
   - `keyboardWeight = 0.15 + keyboardScore * 0.7` (maps [0,1] to [0.15, 0.85])
   - `saxWeight = 1.15 - keyboardWeight` (complementary, also within [0.15, 0.85])

5. Confidence:
   - When normalizedEntropy < 0.3 or > 0.5: confidence = 0.8 (clear signal)
   - When between 0.3 and 0.5: confidence = 0.3 (ambiguous zone)
   - Scale confidence by how non-zero the chroma sum is (if sum < 1, reduce confidence)

6. Return `{ saxWeight, keyboardWeight, confidence }`.
