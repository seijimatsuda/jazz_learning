**Types:**
```typescript
export interface KeyDetectionResult {
  key: string | null;          // e.g. 'C', 'G#', or null if insufficient data
  mode: 'major' | 'minor' | null;
  confidence: number;          // 0.0-1.0, ratio of best pitch class weight to total
}
```

**detectKey function (KEY-01):**
```typescript
export function detectKey(
  chordLog: ReadonlyArray<{ audioTimeSec: number; chordIdx: number; confidenceGap: number }>,
  currentTimeSec: number,
  windowSec: number = 30,
): KeyDetectionResult
```

Algorithm:
1. Pre-allocate `rootWeight = new Float32Array(12)` (no per-call allocation outside this function — but this is called at 2fps max so one allocation is fine).
2. Filter chordLog to entries where `audioTimeSec >= currentTimeSec - windowSec`.
3. For each entry in the window:
   - Look up `CHORD_TEMPLATES[entry.chordIdx]` to get `.root` and `.type`
   - Find `rootIdx = NOTE_NAMES.indexOf(tmpl.root)`
   - Add `entry.confidenceGap` to `rootWeight[rootIdx]`
4. Find the pitch class with the highest cumulative weight (`bestIdx`).
5. If `rootWeight[bestIdx] === 0`, return `{ key: null, mode: null, confidence: 0 }`.
6. Compute confidence: `rootWeight[bestIdx] / totalWeight` where totalWeight is sum of all rootWeight entries.
7. Determine mode: look at all entries in the window whose root matches `bestIdx`. Sum confidenceGap into `majorW` and `minorW`:
   - Major types: 'major', 'maj7', 'dom7'
   - Minor types: everything else ('minor', 'm7', 'dim7', 'm7b5', 'alt')
   - If `majorW >= minorW` → 'major', else 'minor'
8. Return `{ key: NOTE_NAMES[bestIdx], mode, confidence }`.

**chordFunctionInKey function (KEY-02):**
```typescript
export function chordFunctionInKey(
  chordRoot: string,
  chordType: string,
  keyRoot: string,
  keyMode: 'major' | 'minor',
): string
```

Algorithm:
1. Define `INTERVAL_TO_DEGREE` map:
   ```
   { 0: 'I', 1: 'bII', 2: 'II', 3: 'bIII', 4: 'III', 5: 'IV', 6: '#IV/bV', 7: 'V', 8: 'bVI', 9: 'VI', 10: 'bVII', 11: 'VII' }
   ```
2. Compute `interval = ((NOTE_NAMES.indexOf(chordRoot) - NOTE_NAMES.indexOf(keyRoot)) % 12 + 12) % 12`.
3. Look up degree from INTERVAL_TO_DEGREE.
4. Build chord display name: `chordRoot` + type suffix. Map chord types to display suffixes:
   - 'major' → '', 'minor' → 'm', 'maj7' → 'maj7', 'm7' → 'm7', 'dom7' → '7', 'dim7' → 'dim7', 'm7b5' → 'm7b5', 'alt' → 'alt'
5. Return string: `"${chordDisplayName} is the ${degree} chord in ${keyRoot} ${keyMode}"`
   - Example: "G7 is the V chord in C major"

**Edge cases to handle:**
- If chordLog is empty or all entries are outside the window, return null key.
- If chordIdx is out of range for CHORD_TEMPLATES, skip that entry (guard with `if (entry.chordIdx < 0 || entry.chordIdx >= CHORD_TEMPLATES.length) continue`).
- If keyRoot or chordRoot is not in NOTE_NAMES, return a fallback string.
