**VisualizerCanvas.tsx — add click handler:**

After the ResizeObserver setup inside the useEffect, add a click handler on the canvas element:

```typescript
const handleCanvasClick = (e: MouseEvent) => {
  const renderer = rendererRef.current;
  if (!renderer) return;

  const { positions, width, height } = renderer.getNodeLayout();
  const rect = canvas.getBoundingClientRect();
  const clickX = e.clientX - rect.left;
  const clickY = e.clientY - rect.top;
  // Convert CSS pixels to fractional canvas coordinates
  const fx = clickX / rect.width;
  const fy = clickY / rect.height;

  const hitRadius = 0.06; // ~48px on 800px canvas
  for (let i = 0; i < positions.length; i++) {
    const dx = fx - positions[i].x;
    const dy = fy - positions[i].y;
    if (Math.sqrt(dx * dx + dy * dy) < hitRadius) {
      useAppStore.getState().setSelectedInstrument(INSTRUMENT_ORDER[i]);
      return;
    }
  }
  // Clicked outside all nodes — close detail panel
  useAppStore.getState().setSelectedInstrument(null);
};

canvas.addEventListener('click', handleCanvasClick);
```

Use a static import for INSTRUMENT_ORDER (not require). Add it to the existing import from NodeLayout at the top of the file:
```typescript
import { INSTRUMENT_ORDER } from '../canvas/nodes/NodeLayout';
```

Add to the cleanup function:
```typescript
canvas.removeEventListener('click', handleCanvasClick);
```

**Create NodeDetailPanel.tsx:**

A React component that renders the instrument detail panel when an instrument is selected.

**Imports:**
```typescript

**Props:**

interface NodeDetailPanelProps {

}
```

**Component logic:**
1. Read `selectedInstrument` from Zustand. If null, return null (don't render).
2. Read `instrumentRoles` from Zustand for the current role badge.
3. Use two `useRef<HTMLCanvasElement>` for sparkline and pie mini-canvases.
4. Use a `setInterval(100)` to poll `audioStateRef.current.analysis` for the selected instrument's data:
   - `InstrumentAnalysis.historyBuffer` / `historyHead` / `historySamples` for sparkline
   - `InstrumentAnalysis.timeInRole` for pie chart
   - `AnalysisState.edgeWeights` for most-active partner
5. On each poll, draw the sparkline and pie chart onto their respective mini-canvases. Also update a local state for `mostActivePartner`.

**Sparkline canvas (UI-05):**
- Size: 200px wide x 60px tall (CSS), with HiDPI backing store.
- On mount and resize, set `canvas.width = 200 * dpr; canvas.height = 60 * dpr; ctx.scale(dpr, dpr)`.
- Read ring buffer in chronological order: `idx = ((head - samples + i) % 100 + 100) % 100`.
- Draw as a line chart: moveTo first point, lineTo subsequent points.
- X axis: `i / (samples - 1) * 200`; Y axis: `60 - historyBuffer[idx] * 60`.
- Stroke color: `'#818cf8'` (indigo-400), lineWidth 1.5.
- Clear before each draw.

**Pie chart canvas (UI-06):**
- Size: 80px x 80px (CSS), with HiDPI backing store.
- Read `timeInRole: Record<RoleLabel, number>`.
- Role colors: soloing='#f59e0b', comping='#3b82f6', holding='#6b7280', silent='#374151'.
- Draw arc segments proportional to each role's cumulative time.
- Start angle: -PI/2 (top of circle). Radius: 35. Center: (40, 40).
- If total time is 0, draw a full gray circle.

**Most-active partner (UI-07):**
- Read `edgeWeights` from `audioStateRef.current.analysis`.
- For the selected instrument, find the key in edgeWeights that includes this instrument name and has the highest weight.
- Key format: alphabetical pair `'bass_drums'`, `'bass_guitar'`, etc.
- Display: "Most active with: {partner name}" or "No active partners" if all weights < 0.3.

**Layout:**
- Dark card style: bg '#13131f', border '1px solid rgba(99,102,241,0.3)', rounded-xl, p-4.
- Top row: instrument name (capitalized, bold, white) + role badge (colored pill matching role colors) + close button (X, top-right).
- Below: sparkline canvas with "Activity (10s)" label.
- Below: pie chart + role time percentages as text.
- Below: "Most active with: {partner}".
