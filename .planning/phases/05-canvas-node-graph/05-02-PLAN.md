Update CanvasRenderer.ts render loop to use role-based node drawing:

1. **Import** `drawNode`, `getRoleRadius`, `getRoleFillColor` from `./nodes/drawNode`.

2. **In the instrument node rendering loop** (from 05-01), replace the placeholder circle drawing with:
   - Look up the instrument's current role from `state.analysis?.instruments`. Find the matching `InstrumentAnalysis` by `instrument` name. If analysis is null or instrument not found, default to `'silent'`.
   - Compute target radius: `const targetRadius = getRoleRadius(role)`
   - Update animState: `animState.baseRadius = targetRadius`
   - Smoothly transition: `animState.currentRadius = lerpExp(animState.currentRadius, targetRadius + animState.radiusNudge, 0.15, deltaMs)` — smooth role transitions over ~200ms
   - Get fill color: `const fillColor = getRoleFillColor(role)`
   - Call `drawNode(ctx, x, y, animState.currentRadius, fillColor, instrument)`

3. **Pass `deltaMs`** to the node rendering section (it should already be available from 05-01's refactor).

4. **Capitalize label display** — pass instrument name with first letter capitalized (e.g., 'Bass', 'Drums', 'Keyboard', 'Guitar') to drawNode.
