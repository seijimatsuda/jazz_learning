  <name>Task 1: Create AudioEngine and useAudioRef hook</name>
  <files>
    src/audio/AudioEngine.ts
    src/hooks/useAudioRef.ts
  </files>
  <action>
    Create src/hooks/useAudioRef.ts:
    - Returns a stable `useRef<AudioStateRef>` initialized with `createInitialAudioState()`
    - This ref is the single source of truth for all audio state
    - NEVER triggers React re-renders — components that need audio data poll via rAF or setInterval

    Create src/audio/AudioEngine.ts with two exported functions:

    **createAudioContext():**
    - Uses `window.AudioContext || (window as any).webkitAudioContext` for iOS fallback
    - Passes `{ sampleRate: 44100 }` to constructor (iOS may ignore this)
    - After creation, reads back `audioCtx.sampleRate` and logs it — this is the authoritative value
    - If `audioCtx.state === 'suspended'`, calls `await audioCtx.resume()`
    - Registers `statechange` event listener: if state becomes `'interrupted'` (iOS background), set a flag. On next user gesture, call `resume()`
    - Returns the AudioContext

    **decodeAudioFile(audioCtx: AudioContext, file: File): Promise<AudioBuffer>:**
    - Uses `file.arrayBuffer()` to get raw bytes
    - Calls `audioCtx.decodeAudioData(arrayBuffer)`
    - Returns the decoded AudioBuffer
    - Throws descriptive error if decode fails (e.g., unsupported format)

    CRITICAL iOS requirements:
    - createAudioContext MUST be called from within a user gesture handler (click/touchend). It must NOT be called on component mount, useEffect, or page load.
    - Use 'click' event, NOT 'touchstart' (touchstart may be a scroll gesture on iOS)
  </action>
  <verify>
    Run `npx tsc --noEmit` — zero errors.
    Verify AudioEngine.ts exports createAudioContext and decodeAudioFile.
    Verify useAudioRef.ts exports useAudioRef hook.
  </verify>
  <done>
    AudioEngine.ts creates AudioContext with iOS fallback, reads back sampleRate, handles suspended/interrupted states. decodeAudioFile reads File to ArrayBuffer and decodes via Web Audio API. useAudioRef hook returns stable ref to AudioStateRef.
  </done>

      6. Stores buffer in `audioStateRef.current.transport.buffer`
      7. Stores duration in `audioStateRef.current.transport.duration`
      8. Calls `useAppStore.getState().setFile(file.name, buffer.duration)` to update UI state
    - Shows loading state during decode ("Decoding...")
    - Shows file name after successful load
    - Shows error message if decode fails

    - Maintain dark background (#0a0a0f), min-h-screen

    IMPORTANT: The file input click and AudioContext creation MUST happen in the same synchronous call stack from the user's click. Do not wrap in setTimeout or async before the AudioContext constructor call.
