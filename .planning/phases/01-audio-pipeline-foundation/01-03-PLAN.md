  <name>Task 1: Create FrequencyBandSplitter with hzToBin</name>
  <files>
    src/audio/FrequencyBandSplitter.ts
  </files>
  <action>
    Create src/audio/FrequencyBandSplitter.ts with the following exports:

    **hzToBin(hz: number, sampleRate: number, fftSize: number): number**
    - Formula: `Math.round(hz * fftSize / sampleRate)`
    - Clamp result to [0, fftSize/2 - 1] to prevent out-of-bounds access
    - CRITICAL: sampleRate parameter must come from audioCtx.sampleRate (read-back value). Never pass a hardcoded constant.

    **buildDefaultBands(sampleRate: number, fftSize: number): FrequencyBand[]**
    - Returns array of FrequencyBand objects with bin indices computed via hzToBin:
      - bass: 20-250 Hz
      - drums_low: 60-300 Hz (kick/snare region)
      - mid: 250-2000 Hz (keyboard primary)
      - mid_high: 300-3000 Hz (guitar primary, overlaps keyboard)
      - drums_high: 2000-8000 Hz (cymbals)
      - ride: 6000-10000 Hz (ride cymbal)
    - Each band has: name, lowHz, highHz, lowBin, highBin

    **getBandEnergy(freqData: Uint8Array, band: FrequencyBand): number**
    - Computes average energy across bins [lowBin, highBin]
    - Returns 0.0-1.0 (normalized from Uint8Array 0-255 range)
    - If lowBin >= highBin, return 0 (degenerate band — possible at low sampleRate)

    Log band ranges on creation: "Band 'bass': 20-250 Hz -> bins [X, Y] (sampleRate: Z, fftSize: W)"
  </action>
  <verify>
    Run `npx tsc --noEmit` — zero errors.
    Verify hzToBin(250, 44100, 4096) returns 23 and hzToBin(250, 48000, 4096) returns 21 (different bins for different sample rates — proves no hardcoding).
  </verify>
  <done>
    hzToBin converts Hz to FFT bin index using runtime sampleRate. buildDefaultBands produces all instrument frequency bands with computed bin indices. getBandEnergy extracts per-band average energy from frequency data. All functions are pure and testable.
  </done>
