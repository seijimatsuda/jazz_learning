    - Read `window.devicePixelRatio || 1`
    - Get canvas CSS dimensions via getBoundingClientRect
    - Set canvas.width = Math.floor(cssWidth * dpr)

    - Set canvas.style.width and canvas.style.height to CSS dimensions
    - Call ctx.scale(dpr, dpr) so drawing coordinates stay in CSS pixels

         - Demonstrate the offscreen glow pattern is working

    **Create src/audio/TensionHeatmap.ts:**

    Export `computeTensionHeatmap(buffer: AudioBuffer, sampleRate: number): Float32Array`

    This pre-computes a per-second tension value across the entire track using Meyda offline extraction.

    Algorithm:
    1. Get channel data: buffer.getChannelData(0) (or average L+R for stereo)
    2. Compute number of windows: Math.ceil(buffer.duration) (one value per second)
    3. For each second, extract a representative slice (~bufferSize=2048 samples centered on that second)
    4. Call `Meyda.extract('chroma', slice)` to get a 12-element chroma vector
       - IMPORTANT: Pass the correct sampleRate to Meyda via the third argument: `Meyda.extract('chroma', slice, { sampleRate })`
       - If Meyda.extract doesn't accept options in offline mode, set `Meyda.sampleRate = sampleRate` before extraction
    5. Compute a simple tension estimate from chroma:
       - Tension = ratio of energy in tritone intervals (e.g., chroma bins 1,3,6,8,10 vs 0,2,4,5,7,9,11)
       - OR simpler: tension = 1 - max(chroma) (low peak = ambiguous/tense harmony, high peak = clear/resolved)
       - This is a placeholder — Phase 3 will implement proper chord-function-based tension
    6. Store in Float32Array of length = number of seconds
    7. If extraction takes >2 seconds for the full track, log a progress message every 25%

    **Update src/components/Timeline.tsx:**
    - After tension heatmap is computed, render it as a colored bar behind the scrubber
    - Map each second to a horizontal pixel range
    - Color: low tension = blue (#3b82f6), high tension = red (#ef4444), lerp between
    - This is a simple canvas-on-canvas or CSS gradient approach
    - The heatmap renders BEFORE playback starts (visible immediately after file load + calibration)
