    Create src/audio/CalibrationPass.ts:

    **runCalibrationPass(audioStateRef: AudioStateRef): Promise<CalibrationThresholds[]>**

    The calibration pass analyzes the first 3 seconds of audio to determine peak and average energy per frequency band. It uses Meyda offline extraction on the raw buffer data (NOT OfflineAudioContext — avoids doubling memory per RESEARCH.md Pitfall 6).

    Algorithm:
    1. Get the decoded AudioBuffer from audioStateRef.transport.buffer
    2. Extract channel data: buffer.getChannelData(0) (mono, or average L+R for stereo)
    3. Determine how many samples = 3 seconds: Math.min(3 * sampleRate, buffer.length)
    4. Slice the channel data into windows of 4096 samples (matching fftSize)
    5. For each window, use a simple FFT magnitude approximation OR use the raw frequency data approach:
       - Actually, since we don't have an AnalyserNode for offline data, use Meyda.extract() on each slice:
         ```typescript
         import Meyda from 'meyda';
         const features = Meyda.extract(['rms', 'spectralCentroid'], slice);
         ```
       - OR compute per-band energy directly from the raw PCM using a simple approach:
         Apply a DFT via Meyda's powerSpectrum extractor on each slice, then map to bands using hzToBin.
    6. For a simpler approach that avoids Meyda offline complexity: use the real-time AnalyserNode approach:
       - Create a temporary AudioBufferSourceNode
       - Connect to the raw AnalyserNode
       - Play the first 3 seconds at accelerated rate (playbackRate = 1.0 is fine, just stop after 3s)
       - Sample getByteFrequencyData every 50ms (60 samples over 3 seconds)
       - Compute per-band peak and average from those 60 samples
       - Stop and disconnect the temporary source

    RECOMMENDED APPROACH: Use the real-time AnalyserNode method (option 6). It reuses the existing dual AnalyserNode setup, is simpler, and avoids offline Meyda API complexity.

    Implementation for real-time calibration:
    ```
    1. Create temp AudioBufferSourceNode from audioStateRef.transport.buffer
    2. Connect to rawAnalyser (do NOT connect to destination — silent calibration)
    3. Start playback from offset 0
    4. Set up an interval (50ms) that calls rawAnalyser.getByteFrequencyData(rawFreqData)
    5. For each band in audioStateRef.bands, call getBandEnergy to get per-band energy
    6. Track peak and running average per band
    7. After 3 seconds (or buffer end if shorter), stop the source and clear interval
    8. Compute thresholds per band:
       - solo = 0.75 * peak
       - comping = 0.40 * peak
       - holding = 0.10 * peak
    9. Store CalibrationThresholds[] in audioStateRef.current.calibration
    10. Set audioStateRef.current.isCalibrated = true
    ```

    Update Zustand store: call setCalibrating(true) at start, setCalibrating(false) at end.

    Show a console log summary: "Calibration complete: bass peak=X avg=Y, mid peak=X avg=Y, ..."
  </action>

    **Add to AudioEngine.ts** a helper to compute current playback position:
    ```typescript
    export function getCurrentPosition(audioState: AudioStateRef): number {
      const t = audioState.transport;
      if (!t.isPlaying || !audioState.audioCtx) return t.pauseOffset;
      return t.pauseOffset + (audioState.audioCtx.currentTime - t.startTime);
    }
    ```
