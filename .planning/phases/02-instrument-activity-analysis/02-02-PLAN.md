  <action>
    Create src/audio/RoleClassifier.ts with the following exports:

    **classifyRole(activityScore: number, currentRole: RoleLabel, hysteresis?: number): RoleLabel**

    The activity score from computeActivityScore is already normalized to [0,1] relative to calibration peak. So the thresholds map directly:
    - soloing: activityScore >= 0.75
    - comping: activityScore >= 0.40
    - holding: activityScore >= 0.10
    - silent: activityScore < 0.10

    Hysteresis logic (default hysteresis = 0.05):
    - When checking if the instrument should LEAVE its current role, require the score to cross the threshold by MORE than the hysteresis amount in the downward direction.
    - When checking if the instrument should ENTER a higher role, require the score to exceed the threshold (no extra buffer needed for upward transitions).
    - Specifically:
      - If currentRole is 'soloing': stay in 'soloing' until activityScore < 0.75 - hysteresis
      - If currentRole is 'comping': stay in 'comping' until activityScore < 0.40 - hysteresis OR activityScore >= 0.75
      - If currentRole is 'holding': stay in 'holding' until activityScore < 0.10 - hysteresis OR activityScore >= 0.40
      - If currentRole is 'silent': stay in 'silent' until activityScore >= 0.10

    Implementation approach:
    - First check upward transitions (can always move up without hysteresis barrier)
    - Then check downward transitions (must cross threshold - hysteresis to move down)
    - This makes the state machine sticky — it resists leaving its current state

    **updateTimeInRole(timeInRole: Record<RoleLabel, number>, currentRole: RoleLabel, deltaSec: number): void**
    - Adds deltaSec to timeInRole[currentRole]
    - deltaSec is typically 0.1 (100ms between 10fps ticks)
    - Mutates timeInRole in place (no allocation)

    File header comment should explain:
    - The thresholds (0.75/0.40/0.10) correspond to CalibrationPass thresholds normalized by peak
    - Hysteresis prevents flicker at threshold boundaries
    - The 10fps update rate means role transitions have ~100ms minimum latency

    Import RoleLabel from './types'.
