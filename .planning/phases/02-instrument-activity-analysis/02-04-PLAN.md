    Create src/audio/CrossCorrelationTracker.ts with the following exports:

    **CORR_WINDOW constant = 20** (2 seconds at 10fps)

    **pearsonR(histA: Float32Array, headA: number, samplesA: number, histB: Float32Array, headB: number, samplesB: number): number**
    - Computes Pearson correlation coefficient over the most recent min(samplesA, samplesB, CORR_WINDOW) samples
    - Reads backwards from head (most recent first): index = (head - 1 - i + 100) % 100
    - Guards:
      - If n < 2, return 0
      - If denominator (sqrt(denA * denB)) === 0, return 0
      - Clamp result to [-1, 1] as final safety
    - Two-pass: first pass computes means, second pass computes covariance and variances
    - Returns Pearson r in [-1, 1]

    **computeEdgeWeight(r: number): number**
    - If Math.abs(r) < 0.3, return 0 (edge suppression, INST-07)
    - Otherwise return r (preserves sign: positive = co-activating, negative = alternating)

    File header: explains 2-second sliding window, Pearson r over circular buffers, and edge suppression rationale.
  </action>

    **Create src/audio/AnalysisTick.ts:**

    Single exported function:

    **runAnalysisTick(state: AudioStateRef, onRoleChange?: (instrument: string, role: RoleLabel) => void): void**

    This is the 10fps orchestrator. It assumes the caller has already checked the 100ms time gate. Steps:

    1. Guard: if !state.analysis or !state.smoothedFreqData or !state.rawFreqData or !state.rawTimeData or !state.smoothedAnalyser or !state.rawAnalyser or !state.isCalibrated, return early.

    2. Pull fresh FFT data:
       - state.smoothedAnalyser.getByteFrequencyData(state.smoothedFreqData)
       - state.rawAnalyser.getByteFrequencyData(state.rawFreqData)
       - state.rawAnalyser.getByteTimeDomainData(state.rawTimeData)

    3. For each instrument in state.analysis.instruments:
       a. Call computeActivityScore(state.smoothedFreqData, instr.bandNames, state.bands, state.calibration, instr.activityScore)
       b. Store the result as newScore on the instrument
       c. Call pushHistory(instr, newScore)
       d. Call classifyRole(newScore, instr.role)
       e. If role changed:
          - Update instr.role = newRole
          - Update instr.roleSinceSec = state.audioCtx?.currentTime ?? 0
          - console.log('[AnalysisTick] role change:', instr.instrument, newRole)
          - Call onRoleChange?.(instr.instrument, newRole)
       f. Call updateTimeInRole(instr.timeInRole, instr.role, 0.1)

    4. Keyboard/guitar disambiguation (only if both 'keyboard' AND 'guitar' are in the instruments list):
       a. Convert rawTimeData to float in pre-allocated rawTimeDataFloat:
          `for (let i = 0; i < state.fftSize; i++) { state.analysis.rawTimeDataFloat[i] = (state.rawTimeData[i] - 128) / 128; }`
          (rawTimeData is now correctly allocated as fftSize=4096 bytes after the AudioEngine fix above)
       b. Call disambiguate(state.rawFreqData, state.analysis.prevRawFreqData, state.analysis.rawTimeDataFloat, state.sampleRate, state.fftSize)
       c. Find keyboard and guitar InstrumentAnalysis objects
       d. Scale: kb.activityScore *= keyboardWeight; gt.activityScore *= guitarWeight

    5. Cross-correlation for all instrument pairs:
       ```
       for (let a = 0; a < instrs.length; a++) {
         for (let b = a + 1; b < instrs.length; b++) {
           const key = [instrs[a].instrument, instrs[b].instrument].sort().join('_');
           const r = pearsonR(instrs[a].historyBuffer, instrs[a].historyHead, instrs[a].historySamples, instrs[b].historyBuffer, instrs[b].historyHead, instrs[b].historySamples);
           state.analysis.edgeWeights[key] = computeEdgeWeight(r);
         }
       }
       ```

    6. Save current rawFreqData as previous: state.analysis.prevRawFreqData.set(state.rawFreqData)

    Imports: computeActivityScore, pushHistory from InstrumentActivityScorer; classifyRole, updateTimeInRole from RoleClassifier; disambiguate from KbGuitarDisambiguator; pearsonR, computeEdgeWeight from CrossCorrelationTracker; types from types.ts.
