<objective>
Create the keyboard vs guitar disambiguation module that uses zero-crossing rate (ZCR) and hand-rolled spectral flux to weight mid-range activity between keyboard and guitar when both are in the lineup.

Purpose: The mid (250–2000Hz) and mid_high (300–3000Hz) bands overlap significantly. When both keyboard and guitar are in the lineup, raw band energy alone cannot distinguish them. Guitar produces higher ZCR (plucked/strummed transients) and higher spectral flux (sharper attacks) than keyboard (sustained, more tonal). This module computes a weight pair (keyboardWeight, guitarWeight) that sums to 1.0, used to scale each instrument's activity score.

Output: KbGuitarDisambiguator.ts with hand-rolled spectralFlux (NOT Meyda's broken version), Meyda-based ZCR, and a disambiguation function returning weight pair.
</objective>

    **computeSpectralFlux(ampSpectrum: Uint8Array, prevAmpSpectrum: Uint8Array): number**
    - HAND-ROLLED. DO NOT use Meyda.extract('spectralFlux') — it is broken in 5.6.3 (negative index bug, returns 0/NaN).
    - Computes half-wave rectified spectral flux:
      ```
      let flux = 0;
      const len = Math.min(ampSpectrum.length, prevAmpSpectrum.length);
      for (let i = 0; i < len; i++) {
        const diff = ampSpectrum[i] - prevAmpSpectrum[i];
        if (diff > 0) flux += diff;
      }
      return flux;
      ```
    - Returns unbounded positive scalar (higher = more spectral change = more guitar-like)

    **computeZcr(rawTimeDataFloat: Float32Array, sampleRate: number, bufferSize: number): number**
    - Uses Meyda.extract('zcr', rawTimeDataFloat) — verified correct in 5.6.3
    - MUST set Meyda.bufferSize = bufferSize and Meyda.sampleRate = sampleRate BEFORE calling extract
    - rawTimeDataFloat MUST be the pre-allocated buffer from AnalysisState.rawTimeDataFloat (no new allocations)
    - rawTimeDataFloat must already contain float values in [-1, 1] range (converted from Uint8Array before this call)
    - Returns raw ZCR count (integer in [0, bufferSize - 1])

    **disambiguate(rawFreqData: Uint8Array, prevRawFreqData: Uint8Array, rawTimeDataFloat: Float32Array, sampleRate: number, fftSize: number): { keyboardWeight: number; guitarWeight: number }**
    - Calls computeZcr to get raw ZCR
    - Normalizes ZCR: normalizedZcr = zcr / (fftSize - 1) → [0, 1]
    - Calls computeSpectralFlux to get raw flux
    - Normalizes flux: normalizedFlux = Math.min(1, flux / 5000)
      (5000 is a starting constant per research open question #1 — empirical tuning needed later)
    - Computes guitarScore = (normalizedZcr + normalizedFlux) / 2
    - Computes keyboardWeight = 1 - guitarScore, guitarWeight = guitarScore
    - Clamps both weights to [0.15, 0.85] to prevent total suppression of either instrument
      (even with strong disambiguation signal, never fully zero out an instrument)
    - Returns { keyboardWeight, guitarWeight }

    **File header comment must include:**

    - The Uint8Array→Float32Array conversion for Meyda ZCR happens OUTSIDE this module (caller converts using pre-allocated AnalysisState.rawTimeDataFloat)
    - The clamping to [0.15, 0.85] is a safety margin — both instruments always get at least 15% weight
