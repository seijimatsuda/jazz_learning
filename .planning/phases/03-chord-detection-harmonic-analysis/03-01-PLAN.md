    **Private helper: cosineSim(chroma: Float32Array, template: number[]): number**
    Standard cosine similarity over 12 elements. Return 0 if denominator is 0.

    **Private helper: applyBassWeighting(chroma, rawFreqData, sampleRate, fftSize): void**
    - Find loudest bin in bass band (20-250 Hz) using hzToBin
    - Skip if maxEnergy < 20 (no meaningful bass signal)
    - Convert bin -> Hz -> pitch class (semitone % 12)
    - Boost: `chroma[pitchClass] = Math.min(1.0, chroma[pitchClass] * 1.5)`
    - Uses C0 = 16.352 Hz as reference for semitone calculation

    **Private helper: matchChord(chroma: Float32Array): { bestIdx, bestScore, confidenceGap }**
    - Loop over all 96 CHORD_TEMPLATES, compute cosineSim
    - Track best and second-best scores
    - Return bestIdx, bestScore, confidenceGap (best - secondBest)

    **extractAndMatchChord(state: AudioStateRef, audioTimeSec: number): void (exported)**
    The main per-tick function. Steps:
    1. Guard: return if !state.chord or !state.analysis or !state.rawFreqData
    2. Ensure rawTimeDataFloat is populated (it already is from AnalysisTick when kb/guitar disambiguation runs, but for safety, also populate it here if analysis has instruments but none triggered disambiguate):
       - If rawTimeDataFloat is all zeros AND rawTimeData exists, convert:
         `for (let i = 0; i < state.fftSize; i++) { analysis.rawTimeDataFloat[i] = (state.rawTimeData[i] - 128) / 128; }`
    3. Extract chroma: `const chromaRaw = Meyda.extract('chroma', state.analysis.rawTimeDataFloat) as number[];`
    4. Copy into pre-allocated chromaBuffer: `for (let i = 0; i < 12; i++) chord.chromaBuffer[i] = chromaRaw[i];`
    5. Apply bass weighting: `applyBassWeighting(chord.chromaBuffer, state.rawFreqData, state.sampleRate, state.fftSize)`
    6. Update chroma history ring buffer (3 frames):
       - Write current chroma into chromaHistory at offset `chord.chromaHistoryHead * 12`
       - Advance head: `(head + 1) % 3`
    7. Compute smoothed chroma: average of all 3 frames in chromaHistory into smoothedChroma
    8. Match chord: `const match = matchChord(chord.smoothedChroma)`
    9. Update confidence gap: `chord.confidenceGap = match.confidenceGap`
    10. Flicker prevention (CHORD-06):
        - If match.bestIdx === chord.pendingChordIdx, increment pendingHoldCount
        - If pendingHoldCount >= 2 (200ms), update displayedChordIdx and push to chord log
        - Else (new candidate), reset pending to match.bestIdx with holdCount=1
    11. Chord log push (CHORD-11):
        - When displayed chord changes, push { audioTimeSec, chordIdx: match.bestIdx, confidenceGap }
        - If chordLog.length > chordLogMaxLen, splice first entry: `chord.chordLog.splice(0, 1)`
